IA: Good attempt—you’re circling the idea, but let’s tighten it up a bit because this detail really matters for security.

IA: A salt is actually a random value added to the password before hashing, not the hash itself. Its main purpose is to make sure that:

- Two users with the same password don’t end up with the same hash
- Attackers can’t use precomputed tables (like rainbow tables) to crack passwords efficiently

IA: The “10” in genSalt(10) refers to the cost factor (work factor), which controls how computationally expensive the hashing process is—not literally “hashing 10 times,” but increasing the effort required to compute the hash.

IA: So you're partially right about iteration/cost, but it's more about making hashing slower and harder to attack, not repeating it in a simple loop.
